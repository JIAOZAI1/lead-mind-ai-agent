# Go 开发规范（基于 Uber Go Style Guide 整理）

来源：https://github.com/uber-go/guide/blob/master/style.md

---

## 一、指导原则（Guidelines）

### 1. 接口相关
- **不要用指向接口的指针**：接口本身已经是（类型信息 + 数据指针）的组合，传值即可，除非方法需要修改底层数据。
- **编译期校验接口实现**：`var _ InterfaceName = (*Type)(nil)`，对外部导出的类型建议加上，右侧用零值断言。
- **Receiver 一致性**：值接收者的方法可在指针上调用；指针接收者的方法只能在指针或可寻址的值上调用（map 中的值不可寻址）。

### 2. 并发与资源
- **Mutex 零值即可用**：`sync.Mutex`/`sync.RWMutex` 无需指针，作为结构体的非指针字段直接嵌入（但不要作为可导出的嵌入类型暴露）。
- **defer 做清理**：多返回路径下用 defer 保证资源释放，性能开销可忽略，可读性收益更大。
- **Channel 大小只用 1 或不设**：默认无缓冲；使用更大的 buffer 需要经过审慎评审。
- **不要“发后不管”的 goroutine**：每个 goroutine 必须有可预期的退出时机；用 `sync.WaitGroup` 或 `chan struct{}` 等待；禁止在 `init()` 中启动 goroutine；可用 `go.uber.org/goleak` 检测泄漏。

### 3. 枚举与时间
- **枚举从 1 开始**：`iota + 1`，避免与零值（未设置状态）混淆，除非零值本身就是合理默认行为。
- **时间统一用 `time` 包**：`time.Time` 表示时刻，`time.Duration` 表示时长，不要自造整数时间戳/毫秒数处理逻辑。

### 4. 错误处理
- **错误类型选择**：
  | 场景 | 用法 |
  |---|---|
  | 静态消息，无需匹配 | `errors.New` |
  | 动态消息，无需匹配 | `fmt.Errorf` |
  | 静态消息，需要匹配 | 包级 `var` + `errors.New` |
  | 动态消息，需要匹配 | 自定义 error 类型 |
- **错误包装**：用 `fmt.Errorf` + `%w`（保留可匹配性，配合 `errors.Is`/`errors.As`）或 `%v`（不暴露底层错误）；避免多余的 "failed to" 这类冗余措辞。
- **错误命名**：变量用 `Err`/`err` 前缀（`ErrNotFound`/`errNotFound`），自定义类型用 `Error` 后缀（`NotFoundError`）。
- **错误只处理一次**：要么 log，要么 return，不要既 log 又 return，避免日志重复刷屏。
- **类型断言必须用 comma-ok**：`v, ok := x.(T)`，不要用单返回值形式，否则失败会 panic。
- **不要 panic**：生产代码用返回 error 交给调用方处理；例外是程序初始化阶段的不可恢复错误；测试中用 `t.Fatal`，不用 panic。

### 5. 全局状态与初始化
- **避免可变全局变量**：用依赖注入代替修改全局状态，提升可测试性、显式化依赖。
- **避免 `init()`**：如必须使用，需完全确定性、不依赖其他 init 的执行顺序、不访问全局/环境状态、不做 I/O；优先放到 `main()` 或工厂函数中。
- **`os.Exit`/`log.Fatal*` 只能出现在 `main()`**：其他函数一律返回 error；建议 `main()` 里只调用一次 `run()`，方便测试且保证 defer 都能执行。

### 6. API 设计
- **避免在公开结构体中嵌入类型**：会泄露实现细节、锁死类型演进；应手写委托方法。
- **避免使用内建标识符作为名字**（如 `error`、`string`），会造成遮蔽和阅读混乱，用 `go vet` 辅助检测。
- **序列化字段必须打 tag**：如 `json:"fieldName"`，让序列化契约显式化，避免字段改名等意外破坏兼容性。
- **原子操作优先用 `go.uber.org/atomic`**：比 `sync/atomic` 多一层类型安全，内置 `atomic.Bool` 等类型，避免误用导致的竞态。

## 二、性能相关

1. **基础类型转字符串优先用 `strconv` 而非 `fmt`**（热路径下有 2-3 倍提升）。
2. **避免重复的 string↔[]byte 转换**：转一次、复用结果，可减少 6-7 倍的内存分配。
3. **提前指定容器容量**：
   - map：`make(map[K]V, hint)`
   - slice：`make([]T, len, cap)`
   减少扩容和 GC 压力。

## 三、代码风格

- **行长度**：软性建议 99 字符以内。
- **一致性优先**：项目/包内保持统一风格，优先于个人偏好。
- **相关声明分组**：import、const、var、type 用块声明，相关的放一起，不相关的分开。
- **import 分两组**：标准库一组，第三方一组，中间空行（`goimports` 默认行为）。
- **包名**：全小写、简短、单数，不用 `common`/`util`/`shared`/`lib` 这类无意义名字，尽量不需要 import 别名。
- **函数名**：MixedCaps（驼峰），测试函数可用下划线分组（如 `TestFoo_EdgeCase`）。
- **import 别名**：仅当包名与路径最后一段不一致时才用别名。
- **函数排列顺序**：按调用顺序、按 receiver 分组；导出函数在前，`NewXYZ()` 紧跟类型定义之后，工具函数放最后。
- **减少嵌套**：错误/特殊情况优先处理并 `return`/`continue`，保持主逻辑平铺。
- **不必要的 else**：两个分支只是给同一变量赋不同值时，用 `a := default; if cond { a = value }` 代替 if-else。
- **顶层变量声明**：优先 `var x = expr`（省略类型，编译器推导），除非表达式类型与期望类型不同才显式指定。
- **未导出全局变量/常量加 `_` 前缀**（error 变量除外，用 `err` 前缀），让全局作用域在使用处一目了然。
- **结构体嵌入要谨慎**：只有明确功能收益时才嵌入；放在字段最前面并与普通字段分隔；不要嵌入 mutex；判断标准是"内部方法是否都适合暴露在外层类型上"。
- **局部变量声明**：显式赋值用 `:=`；零值场景（如空 slice）用 `var` 更清晰。
- **nil 是合法的 slice**：返回 `nil` 而非 `[]T{}`；判空用 `len(s) == 0` 而不是 `s == nil`。
- **缩小变量作用域**：尽量在靠近使用的地方声明（但不要与"减少嵌套"原则冲突）。
- **避免裸参数（naked parameter）**：不清晰的 bool/易混淆参数要加注释或者干脆用自定义类型替代裸 bool，提升类型安全与可读性。
- **用原始字符串字面量（反引号）**：避免字符串中转义反斜杠、引号。
- **结构体初始化**：
  - 始终用字段名，不要位置初始化；
  - 零值字段不必显式写出；
  - 全零值结构体用 `var s T`，而不是 `s := T{}`；
  - 结构体引用用 `&T{}`，不用 `new(T)`。
- **map 初始化**：尽量带容量提示 `make(map[K]V, capacity)`。
- **格式化字符串**：Printf 之外复用的格式串定义为常量，不要写在调用处。
- **Printf 风格函数命名**：以 `f` 结尾（如 `Errorf`、`Sprintf`）。

## 四、常用模式

1. **表驱动测试（Test Tables）**：多场景测试用一张 test case 表驱动，比写多个独立测试函数更清晰，尤其适合参数组合较多的情况。
2. **函数式选项（Functional Options）**：`func New(opts ...Option) T`，用于可扩展、向后兼容的可选参数 API 设计。

---

## 五、Lint 与工具链

- 代码必须通过 `golint`、`go vet` 检查，无报错。
- 编辑器需配置保存时自动运行 `goimports`。
- CI/本地建议自动化跑 lint 检查，不要人工把关。

## 速查清单（Checklist）

- [ ] 接口传值不传指针
- [ ] 导出类型有编译期接口校验
- [ ] Mutex 非指针字段，零值可用
- [ ] 资源清理用 defer
- [ ] channel 无缓冲或缓冲为 1
- [ ] 枚举从 1 开始
- [ ] 时间用 `time.Time`/`time.Duration`
- [ ] error 类型选择符合场景（New/Errorf/var/自定义类型）
- [ ] 错误只在一处处理（log 或 return，不两者都做）
- [ ] 类型断言用 comma-ok
- [ ] 不 panic（除初始化阶段）
- [ ] 无可变全局变量，依赖显式注入
- [ ] 不在公开结构体里嵌入类型（除非必要）
- [ ] 不用内建标识符做变量名
- [ ] 序列化结构体字段打好 tag
- [ ] goroutine 有明确退出机制，无 fire-and-forget
- [ ] `os.Exit`/`log.Fatal` 只出现在 main()
- [ ] import 分组（标准库/第三方）
- [ ] 包名小写、简短、无 util/common
- [ ] 减少嵌套，错误分支提前 return
- [ ] slice 空值返回 nil，不返回 `[]T{}`
- [ ] 结构体初始化用字段名，零值字段省略
- [ ] 通过 golint/go vet，goimports 自动格式化
